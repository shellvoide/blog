AWS Client VPN is a managed, OpenVPN-based remote access service that lets users securely connect to private AWS networks from anywhere. It is designed for teams that need controlled access to internal workloads without exposing private resources directly to the internet.

In many environments, the challenge is not creating a VPN tunnel, it is managing identity, access scope, and onboarding at scale. That is where integrating AWS Client VPN with IAM Identity Center (SSO) becomes valuable.

This guide walks through the end-to-end setup so users can authenticate with corporate identities, download configuration from the self-service portal, and access only the VPC resources they are authorized to use.

## Why This Approach Matters

Instead of sharing static certificates or manually distributing profiles, this model gives you identity-based access with centralized group control in IAM Identity Center. You can apply least privilege through authorization rules and simplify user onboarding with the self-service portal.

If your team needs auditable, group-driven VPN access that scales cleanly, this is a practical production baseline.

## Who This Guide Is For

## Architecture at a Glance

1. IAM Identity Center hosts two SAML applications:
   - Client VPN authentication app
   - Client VPN self-service portal app
2. IAM receives each app metadata as a separate SAML identity provider.
3. Client VPN endpoint uses those providers for login and self-service.
4. Access is limited by an authorization rule mapped to an Identity Center group.

## Prerequisites

- AWS account with permissions for:
  - AWS IAM Identity Center
  - AWS IAM
  - AWS Client VPN
  - ACM
  - VPC
- A VPC with at least one subnet
- A server certificate in ACM for the Client VPN endpoint
- AWS Client VPN desktop client installed (for end-user testing)

## Step 1: Create SSO App for Client VPN

Go to **AWS IAM Identity Center -> Applications**.

![IAM Identity Center Applications](/static/images/client-vpn-sso/image.png)

Click **Add application**.

![Add Application](/static/images/client-vpn-sso/image-1.png)

Choose **I have an application I want to set up** and select **SAML 2.0**.

![Choose SAML 2.0](/static/images/client-vpn-sso/image-2.png)

![Choose SAML 2.0 Confirmation](/static/images/client-vpn-sso/image-3.png)

Set:

- Application Name: `Client VPN`
- Description: `Application for AWS Client VPN authenticated with SSO`

![Configure Client VPN App](/static/images/client-vpn-sso/image-4.png)

Select your IP stack (IPv4 in this guide) and download the IAM Identity Center SAML metadata file.

![Download Metadata](/static/images/client-vpn-sso/image-5.png)

Set application metadata:

- Application SAML audience:

```text
urn:amazon:webservices:clientvpn
```

- Application ACS URL:

```text
http://127.0.0.1:35001
```

![Client VPN Metadata Values](/static/images/client-vpn-sso/image-6.png)

Leave **Application Start URL** and **Relay State** empty for now, then submit.

![Client VPN App Created](/static/images/client-vpn-sso/image-7.png)

Open **Actions -> Edit attribute mappings** and add:

- Subject -> `\${user:email}` with type `emailAddress`
- `memberOf` -> `\${user:groups}` with type `unspecified`

![Edit Attribute Mappings](/static/images/client-vpn-sso/image-8.png)

![Client VPN Attribute Mappings](/static/images/client-vpn-sso/image-9.png)

Checkpoint: Your `Client VPN` SAML app exists with the two mappings above.

## Step 2: Create SSO App for Self-Service Portal

Create another SAML 2.0 app in IAM Identity Center.

Set:

- Display Name: `Client VPN Self Service Portal`
- Description: `Application for AWS Client VPN self service portal authenticated with SSO`

![Self Service App Config](/static/images/client-vpn-sso/image-10.png)

![Self Service App Config 2](/static/images/client-vpn-sso/image-11.png)

Use the same IP stack (IPv4 in this guide) and download this app's SAML metadata file as well.

Set:

- Application ACS URL:

```text
https://self-service.clientvpn.amazonaws.com/api/auth/sso/saml
```

- Application SAML audience:

```text
urn:amazon:webservices:clientvpn
```

![Self Service Metadata Values](/static/images/client-vpn-sso/image-12.png)

Submit.

![Self Service App Created](/static/images/client-vpn-sso/image-13.png)

Add the same attribute mappings as Step 1:

- Subject -> `\${user:email}` type `emailAddress`
- `memberOf` -> `\${user:groups}` type `unspecified`

![Self Service Edit Mapping](/static/images/client-vpn-sso/image-14.png)

![Self Service Attribute Mapping Values](/static/images/client-vpn-sso/image-15.png)

Checkpoint: Both SAML apps are created and mapped consistently.

![Self Service IAM Provider](/static/images/client-vpn-sso/image-18.png)

## Step 4: Create Client VPN Endpoint

Go to **VPC -> Client VPN Endpoints -> Create Client VPN Endpoint**.

![Create Client VPN Endpoint](/static/images/client-vpn-sso/image-19.png)

Before creating the endpoint, ensure your ACM certificate is ready (public cert for your chosen domain).

Recommended endpoint settings:

- Name tag: `Client VPN Endpoint`
- Description: `AWS Client VPN Endpoint with SSO authentication`
- VPC: your target VPC
- Security group: allow inbound `443`
- Client IPv4 CIDR: `10.200.0.0/20`
- Server certificate ARN: your ACM certificate
- Authentication:
  - SAML provider: `AWS-Client-VPN-SSO-Provider`
  - Self-service provider: `AWS-Client-VPN-Self-Service-Portal-SSO-Provider`
- Client interface options:
  - Enable self-service portal
- DNS: default (for this guide)
- Protocol:
  - OpenVPN
  - Split tunnel enabled
  - UDP over `443`
- Session monitoring:
  - 8-hour timeout
  - Disconnect on session timeout enabled

## Step 5: Associate Target Network

Select the endpoint, open **Associations**, then click **Associate Target Network**.

![Associate Target Network Menu](/static/images/client-vpn-sso/image-27.png)

Choose a subnet in the same VPC as the endpoint.

![Select Subnet Association](/static/images/client-vpn-sso/image-28.png)

It can take about 10 minutes to become `available`.

Checkpoint: At least one subnet association is active.

## Step 6: Create Group and Authorization Rule

In IAM Identity Center, create a group for VPN users.

![Create Group Start](/static/images/client-vpn-sso/image-29.png)

![Create Group Details](/static/images/client-vpn-sso/image-30.png)

Open the group and copy its group ID.

![Copy Group ID](/static/images/client-vpn-sso/image-31.png)

Back in Client VPN endpoint, open **Authorization rules -> Create authorization rule**.

![Create Authorization Rule](/static/images/client-vpn-sso/image-32.png)

Set:

- Destination network: your VPC CIDR
- Grant access: `Allow access to users in a specific access group`
- Access group ID: the group ID copied above

![Authorization Rule Values](/static/images/client-vpn-sso/image-33.png)

Checkpoint: Only users in the selected Identity Center group can access VPC networks.

## Step 7: Assign Applications to Group

From endpoint details, copy the self-service portal URL.

![Copy Self Service URL](/static/images/client-vpn-sso/image-34.png)

Go to the **Client VPN Self Service Portal** app and set:

- Application Start URL: the copied self-service portal URL

![Set App Start URL](/static/images/client-vpn-sso/image-35.png)

Now assign both apps to the VPN group:

- `Client VPN`
- `Client VPN Self Service Portal`

![Assign Users and Groups](/static/images/client-vpn-sso/image-36.png)

Users in that group will now see both apps in the IAM Identity Center access portal.

![Apps Visible to Users](/static/images/client-vpn-sso/image-37.png)

Checkpoint: Group assignment completed for both applications.

## Step 8: End-User Connection Flow

User opens the **Client VPN Self Service Portal** app and downloads the client configuration.

![Open Self Service Portal](/static/images/client-vpn-sso/image-38.png)

In AWS Client VPN desktop app:

1. Open **Manage Profiles**
2. Add profile
3. Upload downloaded config file
4. Save and connect

![Add VPN Profile](/static/images/client-vpn-sso/image-39.png)

Browser-based SSO opens automatically. After successful sign-in, VPN connects.

![Connected Session](/static/images/client-vpn-sso/image-40.png)

Test access to private resources (for example, ping a private IP in the VPC).

Checkpoint: End-user can authenticate via SSO and access private VPC resources.

## Troubleshooting

### Users cannot see applications in IAM Identity Center portal

- Confirm group assignment on both apps.
- Confirm users are members of the assigned group.
- Re-check app assignment propagation (can take a few minutes).

### SSO login works but VPN does not connect

- Verify endpoint is in `available` state.
- Verify target network association is complete.
- Confirm security group allows required traffic.
- Confirm local network/firewall is not blocking UDP 443.

### Connected but cannot access VPC resources

- Validate authorization rule destination CIDR.
- Verify route tables and NACLs in target subnet path.
- Check resource security groups allow client CIDR range.
- Confirm split-tunnel behavior matches expected routes.

## FAQ

**Do I need two SAML apps?**

Yes. One is for Client VPN authentication and one is for the self-service portal.

**Can I use a different client CIDR?**

Yes, as long as it does not overlap with existing VPC/on-prem ranges.

**Do I have to use self-service portal?**

It is optional but strongly recommended for easier user onboarding and config distribution.

## Final Notes

This setup gives you identity-based VPN access with centralized user and group management. For production, consider tightening security groups, adding monitoring/alerts, and documenting your network access policy for each user group.
